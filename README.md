# Web-Форум

## Быстрый старт

```bash
# Создать и наполнить БД
gradle cleanDb createDb initDb

# Запустить приложение на тех же данных
gradle appRun

# Запустить модульные и интеграционные тесты
gradle test jacocoTestReport

# Собрать WAR
gradle war

# Запустить системные тесты
gradle systemTest
```

Отчёты:
- `build/reports/tests/test/index.html` — результаты интеграционных тестов
- `build/reports/tests/systemTest/index.html` — результаты системных тестов
- `build/reports/jacoco/test/html/index.html` — покрытие кода

Тестовые учётные записи из `init.sql`:
- `admin` / `admin`
- `moderator1` / `moderator1`
- `ivan` / `password`
- `anna` / `password`


## Описание сценариев системного тестирования

Системные тесты реализованы в файле `test/SeleniumTest.java` с помощью
**Selenium WebDriver** и браузера **Chromium**. Каждый тест открывает
страницу в реальном браузере, заполняет формы, нажимает кнопки и
проверяет содержимое страницы. Если окружение запущено без графического
дисплея, тесты автоматически переключаются в headless-режим; это можно
переопределить флагом `-Dui.headless=false`.

Перед запуском сервер автоматически поднимается на встроенном Tomcat
(порт 8081, in-memory HSQLDB). Тесты выполняются в фиксированном
порядке, моделируя последовательные действия пользователя.

### Просмотр форума гостем

**Сценарий 1. Гость открывает главную страницу**

Предусловие: категорий в БД нет.  
Действие: открыть `GET /forum/`.  
Ожидаемый результат: заголовок страницы «Форум», текст «Разделов пока нет» виден на странице.  
Метод теста: `testGuestViewsEmptyHomePage`

**Сценарий 2. Страница входа доступна**

Предусловие: —  
Действие: открыть `GET /forum/user/login`.  
Ожидаемый результат: форма с полями `username` и `password` присутствует на странице.  
Метод теста: `testLoginPageAccessible`

### Регистрация нового пользователя

**Сценарий 3. Пароли не совпадают**

Предусловие: —  
Действие: заполнить форму регистрации, в поле «Подтверждение» ввести другой пароль, нажать «Зарегистрироваться».  
Ожидаемый результат: страница регистрации показывает ошибку «не совпадают».  
Метод теста: `testRegisterFailPasswordMismatch`

**Сценарий 4. Успешная регистрация**

Предусловие: имя и e-mail не заняты.  
Действие: заполнить форму корректными данными, нажать «Зарегистрироваться».  
Ожидаемый результат: страница входа показывает «Регистрация выполнена».  
Метод теста: `testRegisterSuccess`

**Сценарий 5. Имя пользователя занято**

Предусловие: пользователь с таким именем уже зарегистрирован (сц. 4).  
Действие: повторно заполнить форму с тем же именем, нажать «Зарегистрироваться».  
Ожидаемый результат: страница регистрации показывает ошибку «уже занято».  
Метод теста: `testRegisterFailUsernameExists`

**Сценарий 6. E-mail уже зарегистрирован**

Предусловие: пользователь с таким e-mail уже зарегистрирован (сц. 4).  
Действие: заполнить форму с другим именем, но тем же e-mail, нажать «Зарегистрироваться».  
Ожидаемый результат: страница регистрации показывает ошибку «уже зарегистрирован».  
Метод теста: `testRegisterFailEmailExists`

### Вход в систему

**Сценарий 7. Неверный пароль**

Предусловие: пользователь зарегистрирован (сц. 4).  
Действие: ввести правильное имя и неверный пароль, нажать «Войти».  
Ожидаемый результат: страница входа показывает ошибку «Неверное».  
Метод теста: `testLoginFailWrongPassword`

**Сценарий 8. Успешный вход**

Предусловие: пользователь зарегистрирован (сц. 4).  
Действие: ввести правильные имя и пароль, нажать «Войти».  
Ожидаемый результат: открывается главная страница, в навбаре видно имя пользователя.  
Метод теста: `testLoginSuccess`

### Управление разделами и темами

**Сценарий 9. Администратор создаёт раздел**

Предусловие: вход как `admin`/`admin`.  
Действие: перейти на страницу создания раздела, заполнить название, нажать «Создать раздел».  
Ожидаемый результат: главная страница показывает созданный раздел.  
Метод теста: `testAdminCreatesCategory`

**Сценарий 10. Гость не может создать тему**

Предусловие: раздел существует (сц. 9), пользователь не вошёл.  
Действие: перейти на страницу создания темы.  
Ожидаемый результат: браузер перенаправлен на страницу входа.  
Метод теста: `testUnauthenticatedCannotCreateTopic`

**Сценарий 11. Авторизованный пользователь создаёт тему**

Предусловие: раздел существует (сц. 9), пользователь вошёл.  
Действие: перейти в раздел, заполнить форму создания темы, нажать «Создать тему».  
Ожидаемый результат: открывается страница созданной темы с её заголовком.  
Метод теста: `testCreateTopic`

### Добавление сообщений

**Сценарий 12. Гость видит тему, но не форму добавления**

Предусловие: тема существует (сц. 11), пользователь не вошёл.  
Действие: открыть страницу темы.  
Ожидаемый результат: форма «Добавить сообщение» не отображается; присутствует подсказка «Войдите».  
Метод теста: `testUnauthenticatedCannotPost`

**Сценарий 13. Авторизованный пользователь добавляет сообщение**

Предусловие: тема существует (сц. 11), пользователь вошёл.  
Действие: открыть тему, ввести текст сообщения, нажать «Отправить».  
Ожидаемый результат: текст сообщения и имя автора видны на странице темы.  
Метод теста: `testCreatePost`

### Поиск

**Сценарий 14. Поиск находит существующую тему**

Предусловие: тема существует (сц. 11).  
Действие: перейти на страницу поиска, ввести название темы, нажать «Найти».  
Ожидаемый результат: в результатах поиска присутствует название темы.

### Панель администратора

**Сценарий 15. Администратор видит список пользователей**

Предусловие: вход как `admin`/`admin`.  
Действие: перейти на `/forum/admin/users`.  
Ожидаемый результат: страница содержит имена всех пользователей.  
Метод теста: `testAdminViewsUserList`

**Сценарий 16. Обычный пользователь получает отказ**

Предусловие: вошёл обычный пользователь (роль `user`).  
Действие: перейти на `/forum/admin/users`.  
Ожидаемый результат: страница содержит «403» или «запрещён».  
Метод теста: `testRegularUserCannotAccessAdminPanel`
